# 代码质量检查

## 原则
- 优先遵循项目现有规范与架构。
- 只报告高影响问题：可能导致 Bug、安全风险、数据错误或严重维护障碍。
- 不纠结无关紧要的风格差异。

## 检查清单
1. 命名与可读性
2. 重复代码、硬编码、魔法数字
3. 异常处理、边界条件、空值处理
4. 函数/类/模块的单一职责
5. 耦合、循环依赖、层次混乱
6. 可扩展性与可维护性
7. 性能、安全、资源使用
8. 反模式与不合理设计
9. 格式与注释（优先读项目配置；无则 2 空格、LF、≤120 行宽；注释说“为什么”；禁注释掉的代码；TODO 标责任人）

## 输出
按严重度分为三类，每类格式：`位置 | 问题 | 建议`

- **错误**：Bug、安全、数据风险。
- **警告**：可读性、架构、扩展性问题。
- **建议**：可选优化。

无问题时回复：“未发现明显质量问题。”
