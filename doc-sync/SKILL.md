# 文档同步

代码变更后，同步公共接口、配置、模型和关键流程的变化，仅保留最新状态。

原则：
- 新增或修改 → 更新对应描述。
- 已移除内容 → 删除相关描述，不留痕迹。
- 保持现有文档结构和风格，仅修改受影响部分。
- 优先更新关联文档和 AI 知识文件。

关注：
- README、docs/、API 文档、配置说明、设计文档
- AI 知识文件（如 `.aidocs/`、`specs/` 等）

流程：
1. 根据代码变更识别影响范围。
2. 找出需新增、修改或删除的文档内容。
3. 同步对应文件，确保各处一致。
4. 无需更新则直接结束。

输出：
- 更新：`文件路径 | 更新摘要`
- 删除：`文件路径 | 删除内容`
- 无影响：“本次变更无需更新文档。”
